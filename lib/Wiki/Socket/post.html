Conectar-se usando cURL

Para interagir com o servidor, você pode utilizar cURL para enviar requests POST.

Exemplo de envio de mensagem com Request POST e cURL

Aqui está um exemplo de como enviar um request POST com cURL, utilizando um payload JSON:

curl --insecure -X POST \
  -H "Content-Type: application/json" \
  -d '{
    "username": "seu_username",
    "password": "sua_senha",
    "code": "Código JS para executar",
    "chatId": "id_do_local_para_enviar_mensagem",
    "quoted": false,
    "message": {
      "text": "Oi!"
    }
  }' \
  https://exemplo:8080/send

O parâmetro code será executado apenas se a permissão de execução estiver configurada corretamente no arquivo config.json. Se você não tem certeza do que isso significa ou está em um ambiente inseguro, NÃO ATIVE ESSA CONFIGURAÇÃO, pois é EXTREMAMENTE PERIGOSA.

Se o parâmetro code for enviado, os outros campos do JSON serão ignorados e não serão processados.

O parâmetro quoted pode conter dados de menção a uma mensagem específica. Se não estiver certo, utilize false ou consulte a estrutura do parâmetro quoted na endpoint messages através da conexão websocket.

O uso de --insecure é necessário se estiver conectando via localhost ou um IP de rede privada, pois nestes casos o certificado de conexão é autoassinado e a conexão falhará sem o parâmetro --insecure.

Substitua exemplo:8080 pela URL e porta corretas, que são fornecidas ao iniciar o serviço. Certifique-se de substituir seu_username e sua_senha pelos valores apropriados do seu arquivo config.json. O parâmetro chatId pode ser obtido olhando a página de logs ou o terminal.
